# Slides 审阅检查清单

## 一、内容检查

### 完整性
- [ ] 核心论点是否都已覆盖？
- [ ] 逻辑链条是否完整？
- [ ] 是否有必要的背景介绍？
- [ ] 结论是否清晰明确？

### 准确性
- [ ] 数据和事实是否正确？
- [ ] 引用是否准确标注？
- [ ] 术语使用是否一致？

### 简洁性
- [ ] 是否有冗余内容可删除？
- [ ] 每个要点是否足够精简？
- [ ] 是否用最少的词表达了最多的意思？

## 二、格式检查

### Marp语法
- [ ] YAML frontmatter 是否完整？
- [ ] `---` 分页符是否正确？
- [ ] Header/Footer 是否设置？
- [ ] 缩进是否一致（2空格或4空格）？

### Markdown格式
- [ ] 标题层级是否正确（h1→h2→h3）？
- [ ] 列表格式是否统一（无序/有序）？
- [ ] 粗体和引用是否正确使用？
- [ ] 代码块语言是否标注？

## 三、密度检查（重点）

### 文字数量
| 检查项 | 安全值 | 警告值 | 危险值 | 状态 |
|--------|--------|--------|--------|------|
| 每页文字行数 | ≤6 行 | 7 行 | ≥8 行 | |
| 每行字符数 | ≤35 字符 | 36-40 字符 | >40 字符 | |
| 列表项数量 | ≤5 项 | 6 项 | >6 项 | |
| 嵌套列表深度 | 1 层 | 2 层 | >2 层 | |
| 段落长度 | ≤2 行 | 3 行 | >3 行 | |

> 注意：数学公式按 2 行计算。

### 内容密度分级

**绿色（通过）**：
- 内容清晰，留白充足
- 视觉焦点明确
- 阅读轻松

**黄色（警告）**：
- 内容略多但可接受
- 建议优化但非必须
- 边界案例

**红色（必须修改）**：
- 文字明显过多
- 可能溢出页面
- 必须拆分或精简

## 四、视觉检查

### 层次感
- [ ] 标题是否突出？
- [ ] 重点是否用粗体标记？
- [ ] 引用是否使用blockquote？
- [ ] 信息层次是否清晰？

### 平衡感
- [ ] 内容是否居中或合理分布？
- [ ] 上下左右是否有足够边距？
- [ ] 视觉重心是否稳定？

### 一致性
- [ ] 字体和样式是否统一？
- [ ] 列表符号是否一致？
- [ ] 页面风格是否协调？

## 五、PNG检查专项

### 图片渲染检查
- [ ] 嵌入图片是否正确渲染（非空白）？
- [ ] 使用 PDF→PNG 两步法导出（禁止 `--images png`）？
- [ ] 图片 `max-height` 是否已约束？

### 溢出检查
- [ ] 文字是否被截断？
- [ ] 内容是否超出可视区域？
- [ ] 图片是否溢出 slide 边界？
- [ ] 页脚是否正常显示？

### 元素重叠检查
- [ ] 文字与文字之间无重叠？
- [ ] 文字与图片之间无重叠？
- [ ] 内容密度 ≤8 行（含公式按 2 行计）？

### 渲染检查
- [ ] 中文是否正确显示？
- [ ] 特殊字符是否渲染正确？
- [ ] 代码高亮是否正常？

### 布局检查
- [ ] 各元素位置是否正确？
- [ ] 间距是否合理？
- [ ] 整体布局是否美观？

## 六、问题处理指南

### 溢出问题
```
问题：文字超出页面
解决：
1. 拆分为多页
2. 精简文字内容
3. 使用更简洁的表达
```

### 密度过高
```
问题：单页内容过多
解决：
1. 删除次要信息
2. 将详情移至附录
3. 使用子页面展开
```

### 层次不清
```
问题：信息平铺无重点
解决：
1. 使用粗体标记重点
2. 调整标题层级
3. 使用引用块突出关键内容
```

## 七、评分标准

| 评分 | 标准 |
|------|------|
| A (优秀) | 内容精准、格式规范、密度适中、视觉美观 |
| B (良好) | 基本达标，有小问题但不影响展示 |
| C (及格) | 能用但需优化，存在明显可改进空间 |
| D (不及格) | 有严重问题必须修改（如溢出、逻辑断裂） |

## 八、快速检查流程

1. **5秒扫描**：每页快速浏览，感受密度
2. **内容验证**：核对是否覆盖所有要点
3. **格式检查**：确认Markdown语法正确
4. **PNG验证**：转换后逐页检查渲染结果
5. **整体评估**：给出综合评分和修改建议

## 九、迭代审查协议

### 基本要求
- 最少执行 **3 轮** export→review→fix 循环
- 每轮反馈保存到 `03_reviews/round_N/feedback.md`
- 继续迭代直到满足退出条件

### 退出条件
- **通过**：0 个 MAJOR 问题 且 ≤2 个 MINOR 问题（per deck）
- **最大轮次**：5 轮后强制停止，报告剩余问题

### 问题严重度分类

| 严重度 | 定义 | 示例 |
|--------|------|------|
| MAJOR | 影响可读性或专业性 | 文字溢出、图片空白、元素重叠、内容截断 |
| MINOR | 不影响理解但可改进 | 图表文字偏小、间距略紧、对齐微偏 |

### 每轮记录模板

```markdown
## Round N 审查反馈

- 日期：[日期]
- MAJOR 问题数：X
- MINOR 问题数：Y

### MAJOR 问题
| Slide | 问题 | 修复方案 |
|-------|------|----------|

### MINOR 问题
| Slide | 问题 | 备注 |
|-------|------|------|

### 本轮结论
- [ ] 通过（0 MAJOR，≤2 MINOR）
- [ ] 需要下一轮
```
